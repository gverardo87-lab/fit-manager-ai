# FitManager Guide Illustrated Playbook

## When To Use

Use this skill for:

- screenshot-based help chapters;
- annotated walkthroughs;
- before/after visual guides;
- responsive visual help assets.

## Visual Rules

- Every image must support one concrete action.
- Prefer numbered callouts with short labels.
- Keep one primary focus per screenshot.
- Avoid clutter and duplicate annotations.
- Ensure desktop/tablet/mobile variants for high-traffic pages.

## Workflow

1. Select flow and identify key visual checkpoints.
2. Capture clean screenshots with stable UI state.
3. Add callouts for controls the user must interact with.
4. Pair each visual with one action sentence and one expected result.
5. Validate on responsive breakpoints and dark/light compatibility when relevant.

## Annotation Standards

- Numbered markers (1, 2, 3) in reading order.
- Label length target: 3-7 words.
- Use consistent color logic for severity/info.
- Keep typography legible on 390px width.

## Verification Checklist

- each screenshot has a matching textual step;
- no annotation overlaps critical UI controls;
- no sensitive personal/financial data visible;
- callout language matches real UI labels.
