# agents/ - Local Playbooks for Future Agent Profiles

This folder is optional for Codex runtime, but useful for project-specific playbooks.

Suggested structure:

- `agents/frontend.md` -> UI/UX, query invalidation, type-sync checks
- `agents/api.md` -> security, ownership, transactions, audit rules
- `agents/release.md` -> build, installer, smoke-test and rollback flow

Use these files as focused operating notes.
Keep them concise, actionable, and versioned with code changes.
